Date 周二 14 八月 2018 By 浮绿水 Category 技术. Tags pelican publishing

redis 数据持久化

  • Redis支持数据得持久化，可以将内存中得数据保存在磁盘中，重启得时候可以再次加载使用。
  • Redis不仅仅支持简单得key-value累心得数据，同时还提供list，set，zset，hash等数据结构得存储。
  • Redis支持数据得备份，即master-slave模式得数据备份。

redis得数据类型

Redis支持二进制案例的 Strings, Lists, Hashes, Sets 及 Ordered Sets 数据类型操作。

redis得优势

  • 性能高 redis读得速度是十一万次每秒 写得速度是八万一千次每秒
  • 具有丰富得数据类型 redis支持二进制案例得字符串 列表 哈希 sets 以及ordered sets（有序集合）等数据类型得操作
  • 具有原子性 要么成功执行要么完全不执行 单个操作是原子性得 多个也支持事物
  • 具有丰富得特性 redis 支持通知 或 key过期等特性

Nginx与uWSGI，Apache三者得区别

Date 周三 25 四月 2018 By 浮绿水 Category 技术. Tags pelican publishing

Nginx

Nginx是一款自由的、开源的、高性能的HTTP服务器和反向代理服务器；同时也是一个IMAP、POP3、SMTP代理服务器；Nginx可以作为一个HTTP服务器进行网站的发布处理，另外Nginx可以作为反向代理进行负载均衡的实现。

应用场景 通常情况下，我们在实际项目操作时，正向代理和反向代理很有可能会存在在一个应用场景中，正向代理代理客户端的请求去访问目标服务器，目标服务器是一个反向单利服务器，反向代理了多台真实的业务处理服务器。

uWSGI

WSGI 的官方定义是，the Python Web Server Gateway Interface。它是一个Gateway，也就是网关。网关的作用就是在协议之间进行转换。

uWSGI是一个Web服务器，提高nginx效率。 它实现了WSGI协议、uwsgi、http等协议。Nginx中HttpUwsgiModule的作用是与uWSGI服务器进行交换。

要注意 WSGI / uwsgi / uWSGI 这三个概念的区分。

WSGI是一种通信协议。 uwsgi同WSGI一样是一种通信协议。 而uWSGI是实现了uwsgi和WSGI两种协议的Web服务器。 uwsgi协议是一个uWSGI服务器自有的协议，它用于定义传输信息的类型（type of information），每一个uwsgi packet前4byte为传输信息类型描述，它与WSGI相比是两样东西 …

SSL与TLS得区别

Date 周一 23 四月 2018 By 浮绿水 Category 技术. Tags pelican publishing

  • SSL,TLS

  • SSL：（Secure Socket Layer，安全套接字层），位于可靠的面向连接的网络层协议和应用层协议之间的一种协议层。SSL通过互相认证、使用数字签名确保完整性、使用加密确保私密性，以实现客户端和服务器之间的安全通讯。该协议由两层组成：SSL记录协议和SSL握手协议。
  • SSL协议位于TCP/IP协议与各种应用层协议之间，为数据通讯提供安全支持。SSL协议可分为两层： SSL记录协议（SSL Record Protocol）：它建立在可靠的传输协议（如TCP）之上，为高层协议提供数据封装、压缩、加密等基本功能的支持。 SSL握手协议（SSL HandshakeProtocol）：它建立在SSL记录协议之上，用于在实际的数据传输开始前，通讯双方进行身份认证、协商加密算法、交换加密密钥等。

  • TLS

  • TLS：(Transport LayerSecurity，传输层安全协议)，用于两个应用程序之间提供保密性和数据完整性。该协议由两层组成：TLS记录协议和TLS握手协议。
  • TLS记录协议用于封装各种高层协议。作为这种封装协议之一的握手协议允许服务器与客户机在应用程序协议传输和接收其第一个数据字节前彼此之间相互认证，协商加密算法和加密密钥。

session与cookie得区别

Date 周一 20 十一月 2017 By 浮绿水 Category 技术. Tags pelican publishing

HTTP协议的特性：

每一次来自于用户浏览器的请求（request）都是无状态的、独立的。通俗地说，就是无法保存用户状态，后台服务器根本就不知道当前请求和以前及以后请求是否来自同一用户。

为了保持连接状态，网站会在用户的浏览器通过Cookie可以保存一些诸如用户名、浏览记录、登录和注销等各种数据。但是这种方式非常不安全，因为Cookie保存在用户的机器上，如果Cookie被伪造、篡改或删除，就会造成极大的安全威胁，因此，通常将Cookie用来保存一些不重要的内容，实际的用户数据和状态还是以Session会话的方式保存在服务器端。

cookie：在本地客户端存储数据，可以发送给服务端

Cookie：指某些网站为了辨别用户身份、进行会话跟踪而储存在用户本地的数据（通常经过加密）。 Cookie是由服务器端生成，发送给客户端浏览器，浏览器会将Cookie的key/value保存，下次请求同一网站时就发送该Cookie给服务器（前提是浏览器设置为启用cookie）。 Cookie可以长久的在客户端保存，如购物车

session ：会话~~在服务端存储数据，安全，存储数据多，但有有效期
