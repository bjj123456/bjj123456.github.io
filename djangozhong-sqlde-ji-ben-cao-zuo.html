django中sql得基本操作

Date 周二 13 三月 2018 By 浮绿水 Category 技术. Tags pelican publishing

在Django中操作数据库的方式有两种，第一种就是使用ORM模型来操作，第二种就是使用原生sql语句操作。

命令行管理工具下输入命令进行测试： python .\manage.py shell ：打开数据库测试的shell工具 from 子应用.models import * ：引入models下的所有类 dir() ：查询结果集 exit() ：退出 cls ：清屏

变量 = 类名.objects.create(字段1='值') ：创建数据 （以变量接收内存中的实例） 变量.save() ：保存到数据库 （操作实例完成后必须使用.save()作用到数据库） 【在我们显示调用save()之前，django不会访问数据库，实例数据只存在于内存中】

Manager(管理器)是每一个模型类所具有的，默认名为objects，通过模型类调用orm数据接口，对数据表进行操作。

增： create()方法创建数据： 变量 = 类名.objects.create(字段1='值') 【也可以通过 变量.save() 进行逐条添加】

查： all() ：查询所有，返回一个数据对象结果集 变量 = 类名.objects.all() 【还可以通过下标[0]，切片[0:2]，第一个first()，最后一个last()进行取值】

filter() ：条件查询，返回一个数据对象结果集 变量 = 类名.objects.filter(age=16) 变量 = 类名.objects.all().filter(age=16) 变量 = 类名.objects.filter(age__gt=16) 【还可以在字段名之后使用双下化线来标明更加详细的字段筛选条件也叫做链式过滤】 模糊查询的参数：

__iexact='asd' ：查询字段中是 asd 的数据 忽略大小写 __istartswith='王' ：查询字段中以 王 开始的数据 忽略大小写

__iendswith='王' ：查询字段中以 王 结尾的数据 忽略大小写 __startswith='王' ：查询字段中以 王 开始的数据

__endswith='王' ：查询字段中以 王 结尾的数据 __contains='王' ：查询字段中包含 王 的数据

__in=[1,3,5] ：查询字段中是 1 3 5 的数据 __lt ：小于

__lte ：小于等于 __gt ：大于

__gte ：大于等于

exclude() ：与filter()相反查询所有不满足条件的，参数同上，返回一个数据对象结果集 变量 = 类名.objects.exclude(age=18) ：查询所有年龄不等于18的数据 变量 = 类名.objects.all().filter(age=16).exclude(id=3)

order_by() ：查询结果的排序，返回一个数据对象结果集 变量 = 类名.objects.order_by('age') 变量 = 类名.objects.all().order_by('-age')

get() ：获取唯一单条数据，返回一条匹配的结果 变量 = 类名.objects.get(id=1)

count() ：相当于len()的使用，返回数据库中对应字段的个数 变量 = 类名.objects.filter(age=20).count() 变量 = 类名.objects.count() 【推荐使用len(变量)而不使用count()因为它提高了sql查询的负担】

values() ：返回的是字典而不是数据实例对象结果集 变量 = 类名.objects.all().values()

改： 获取到对应的数据实例之后，通过‘.’的方式访问数据实例中的属性，进行数据的字段修改 变量 = 类名.objects.get(id=1) 变量.age = 21 变量.save()

删： 类名.objects.get(id=1).delete() 【创建和删除是不需要进行.save()保存的】

一对一： 继承关系并且不能使用反向查询。 a1 = 类1.objects.create(name='小王',age=18) # 创建表1的字段行1 a2 = 类1.objects.create(name='小红',age=19) # 创建表1的字段行2 b1 = 类2.objects.create(关联字段名=a1,gender='男') # 表2中的字段名为 关联字段名_id 关联的是a1对应的id行

b1.关联字段名 = a2 # 修改 关联字段名_id 下的关联id为 a2对应的id行 b1.关联字段名.name # 通过 关联id字段 找到对应表1的字段值

一对多： 关联字段名所关联的数据必须是一个实例，并且可以使用反向查询。 【父表中隐藏了子表属性'小写子表名'】 【把关联表字段设置了null=True，子表创建时不填写此字段就可以直接保存。】 a1 = 类1.objects.create(tname='老师1') # 创建表1的字段行1 a2 = 类1.objects.create(tname='老师2') b1 = 类2.objects.create(sname='学生1',关联字段名=a1) # 关联字段名_id 接收a1字段行实例 b2 = 类2.objects.create(sname='学生2',关联字段名=a2)

_set反向查询的使用： a1.小写子类名_set.all() ：通过 父类 查询其下所属 子类

正向查询： 子类.objects.filter(关联字段名__tname='老师1') ：查找老师1带的学生 【查找的是学生，所以用学生表开头，条件是老师1，所以filter中正向直接使用关联字段名找到老师1】 反向查询： 父类.objects.filter(小写子类__sname__endswith='1') ：查找以1结尾的学生的老师 【查找的是老师，所以用老师表开头，条件是学生，所以filter中反向可以使用小写学生表__属性】

使用_set与小写表名的区别： _set 前面使用的是 表中的行数据 （针对于数据） 小写表名 前面使用的是 整个表 （针对于表）

多对多： 可以是具有关联，也可以是没有关联，所以不需要明确指定on_delete属性 【多对多外键关系的添加 用到的是add()函数方法】 a1 = 类1.objects.create(tname='老师1') a2 = 类1.objects.create(tname='老师2') b1 = 类2.objects.create(sname='学生1',sage=11) b2 = 类2.objects.create(sname='学生2',sage=12) b3 = 类2.objects.create(sname='学生3',sage=13) b4 = 类2.objects.create(sname='学生4',sage=14)

b1.关联字段名.add(a1,a2) ：给学生1关联两个老师，存储在外键关联表中

正向查询： b1.关联字段名.all() ：查询学生1的所有老师 b1.关联字段名.filter(tname='老师1') ：查询学生1的老师，名字为老师1的数据

类2.objects.filter(关联字段名__name='老师1') ：查询老师1的所有学生

反向查询： a1.小写表名_set.all() ：查询老师1的所有学生

类1.objects.filter(小写表名=b1) ：查询学生1的所有老师 类1.objects.filter(小写表名__sname='学生1')

删除：多对多关联字段的删除要使用 remove 删除的不是数据而是关系 a1.小写表名_set.remove(b1,b2) ：删除外键关系表中的对应关系数据

django中使用原生SQL的几种方式 ：

一：extra:结果集修改器，一种提供额外查询参数的机制

使用ORM模型： 类名.objects.filter(name='小王',age__gt=50)

使用原生sql：
1. 类名.objects.filter(name='小王').extra(where=['age>10']) 2. 类名.objects.extra(select={'变量':'select count(*) from 数据库中的表名'})

二：raw:执行原始sql并返回模型实例

表名.objects.raw('select * from 数据库中的表名') *********暂空

三：直接执行自定义Sql，前两种是依赖于model的，这种方式完全不依赖model,在views.py中写入def函数

from django.db import connection cursor=connection.cursor()

插入操作

cursor.execute("insert into 数据库中的表名(name) values('小王')")

更新操作

cursor.execute('update 数据库中的表名 set name='小小王' where name='小王'')

删除操作

cursor.execute('delete from 数据库中的表名 where name='小小王'')

查询操作

cursor.execute('select * from 数据库中的表名') raw=cursor.fetchone() #返回结果行游标直读向前，读取一条 cursor.fetchall() #读取所有
